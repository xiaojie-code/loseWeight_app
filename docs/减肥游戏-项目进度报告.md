# 减肥拳击游戏 - 项目进度报告

> 更新时间：2026.5.23

---

## 今天干了什么

主要在搞真机测试，发现了一堆问题然后一个个修。

### 修好的

- **UI 页面系统**：按产品文档把 18 个功能页面全部写完了，包括主菜单、角色选择、地区选择、匹配等待、创建/加入房间、结算、排行榜、装扮、个人中心、设置、战绩、训练等
- **首次进入流程**：加了性别选择 → 地区选择 → 首页的引导流程，用 PlayerPrefs 记录是否首次进入
- **对局信息 HUD**：字体从 Microsoft YaHei 改成 Arial（安卓上 YaHei 不存在会导致文字消失），字号加大，位置下移避开状态栏
- **暂停功能**：对局内右上角加了暂停按钮，弹窗有"继续游戏"和"返回大厅"
- **对局结束弹框**：比赛结束后弹出结果弹窗，有再玩一局、返回首页、分享好友三个按钮
- **返回按钮修复**：设置/排行榜等页面的返回按钮之前有时候没反应，修了
- **模型重复问题**：再玩一局时会出现两个模型叠在一起，修了（BuildScene 先清理旧场景）
- **AI 和玩家模型动作分离**：之前 AI 出拳玩家模型也跟着动，现在分开了
- **受击红色闪烁效果**：玩家被打到时全屏红色闪烁
- **出拳检测优化**：冷却时间从 0.45s 降到 0.25s，左右手独立冷却，可以连续出拳
- **摄像头权限**：加了 AndroidManifest.xml，声明摄像头权限
- **摄像头画面显示**：真机上 MediaPipe 的 WebCam 画面能显示了（之前完全黑屏）
- **AppSettings 修复**：发现 AssetLoaderType 一直配错了（Local 模式在真机上直接退出），改成 StreamingAssets
- **MediaPipe CPU 模式**：发现 PoseLandmarkDetectionConfig 在 Android 上默认用 GPU，改成 CPU
- **屏幕调试日志**：加了绿色屏幕日志，方便真机调试不用连电脑
- **竖屏锁定**：ProjectSettings 改成 Portrait，打包后不会横屏

### 还没搞定的

- **摄像头画面方向**：一直在倒，试了很多次，`rot=270, front=true` 的旋转组合还没找到正确的
- **MediaPipe 姿态检测**：真机上 20 秒内没有产生任何姿态帧，Bootstrap 的日志也没出现，不确定卡在哪一步。今天最后加了超时日志，等下次打包看结果

## 当前主要问题

**1. 姿态检测是最大的坑**

MediaPipe Unity Plugin 在 Android 上一直不工作。排查过的原因：
- ~~AssetLoaderType 配错了~~ → 已修
- ~~GPU 模式在 Android 上不支持~~ → 已改 CPU
- Bootstrap 可能卡住了，导致 PoseLandmarkerRunner 永远等待
- 今天加了超时日志，下次打包能看到具体卡在哪

备用方案：写了 `ServerPoseDetector`，可以把摄像头帧发给服务端的 MediaPipe 做推理，但需要手机和电脑在同一个 WiFi，延迟也会高一些。


**2. 模型动作**

AI 和玩家模型动作分离的问题修了，但真机上还没验证（因为姿态检测不工作，玩家模型根本没有动作输入）。

---

## 下一步

1. 打包看 Bootstrap 超时日志，确认 MediaPipe 卡在哪
2. 如果 MediaPipe 还是不行，接 ServerPoseDetector 用服务端推理
3. 姿态检测通了之后，联调出拳 → 模型动作 → 伤害判定的完整链路
