# 大炮射击模块 落地方案（点击版）

> 版本：v1.3　日期：2026-06-22（在 v1.2 基础上结合市面同类爆款补充：章节化、留存与变现、养成扩展，见十一~十三章）
> 说明：本文基于**当前代码现状**与新整理的玩法需求，给出技术落地路径（改造/新增清单、数据结构、里程碑）。
> 玩法已从「体感瞄准」切换为「**点选目标 + 自动开火**」，体感方案以开关旁路形式完整保留。
> 关联文档：`大炮射击-产品方案（客户版）.md`、`大炮射击体感版-实现方案.md`

## 一、背景与定位

- **核心玩法变更**：由「抬手臂瞄准 + 下劈开炮」改为「**手指点选丧尸/怪物 → 大炮自动锁定并持续开火**」。
- **体感保留**：`CannonPoseInput` 整套（One Euro 滤波、控制手锁定、防抖）通过 `CannonGameController.ManualTapControl` 开关旁路保留，置 `false` 即可切回体感，不删除。
- **本文目标**：把新需求（点选自动攻击、升级、积分、丧尸/怪物、限时挑战、25 关卡、连击反馈、结算分享）逐项映射到现有代码，标明「复用 / 改造 / 新增」并给出落地步骤。

## 三、目标玩法总览

### 核心循环

```
选择模式（限时挑战 / 关卡）
   → 进入对局 → 点选目标 → 大炮自动锁定开火 → 击杀
   → 目标消灭后自动锁定最近目标 → 持续清怪
   → 对局结束（时间到 / 全歼通关 / 防线被攻破）
   → 结算（评级 + 积分）→ 看视频领三倍积分 / 再来一局 / 分享微信
   → 用积分永久升级大炮（射速 / 伤害 / 护盾 / 防御值）→ 再次挑战
```

### 两种模式

| 模式 | 目标 | 结束条件 | 节奏 |
|---|---|---|---|
| 限时挑战 | 60 秒内尽量刷高分 | 时间到 / 防线被攻破 | 难度随时间递增 |
| 关卡模式 | 全歼关卡编排内所有丧尸与怪物 | 全歼通关 / 防线被攻破 | 每关固定编排，难度系数递增（共 25 关） |

## 四、功能详述与落地点

### 4.1 目标选择与自动攻击

**需求**：玩家点选攻击目标；目标被消灭后，大炮自动选择**距离最近**的目标继续攻击。

**现状**：`UpdateManualTapControls()` 已实现点选锁定 + 锁定后自动开火；`OnManualBulletArrived` 在击杀后调用 `ClearManualTarget(true)`，会清空目标并提示重新点击。

**落地动作**：
- `ZombieSpawner` 新增 `bool TryFindNearestZombie(Vector2 fromScreenPos, out Zombie target, out Vector2 pos)`：在存活丧尸中按到炮口（或到上一个目标）距离取最近。
- 改 `OnManualBulletArrived` 击杀分支：不再直接 `ClearManualTarget`，而是先尝试 `TryFindNearestZombie`，找到则续锁，找不到再清空并提示「点击丧尸选择目标」。
- 增加「自动接管」开关感知：玩家手动点选其它目标时可随时打断自动续锁（保留现有点选优先级）。
- 边界：目标在飞弹途中离场（被漏/越界）→ 校验 `IsDead/activeInHierarchy` 后重新选最近。

### 4.2 大炮升级系统（局外永久养成）

**需求（已确认）**：用通关积分**永久**升级大炮（局外养成、账号存档，非局内临时强化）。共 **4 条升级线**：射速、伤害、护盾、防御值（后两者构成「防线」）。

**防线机制（核心规则，取代原 3 条命）**：
- 大炮底座前方有一道**城墙式防线**。丧尸/怪物下落到防线处会**停驻并持续攻击防线**（按各自攻击间隔反复造成伤害），被消灭后才消失。
- 防线有两层耐久：**护盾 Shield** 与 **防御值 DefenseHP（城墙本体生命）**。
- 每次攻击扣减顺序：**先扣护盾，护盾归零后扣防御值**；**防御值也归零 → 防线被攻破 → 对局失败**。
- 多个敌人可同时贴在防线持续输出，因此清线速度（射速/伤害）与防线耐久（护盾/防御值）共同决定生死。
- 护盾与防御值各为一条独立升级线，等级越高初始上限越高、越耐打。
- HUD 用盾条 + 血条展示防线当前护盾/防御值（替代原来的 3 颗心）。

**四条升级线**：

| 升级线 | 作用 | 映射 |
|---|---|---|
| 射速 FireRate | 缩短自动开火间隔 | `ManualFireCooldown`（如 0.42s → 逐级降到 0.15s） |
| 伤害 Damage | 每发炮弹伤害 | `Cannonball.Damage`，`Zombie.TakeHit(int)` 按伤害扣血 |
| 护盾 Shield | 防线护盾上限 | 防线初始护盾值（先于防御值被消耗） |
| 防御值 Defense | 防线本体生命上限 | 防线初始 DefenseHP（护盾耗尽后承伤） |

**落地动作**：
- 新增 `CannonUpgradeConfig`（ScriptableObject）：每条线分等级，定义 等级→数值、等级→升级所需积分。
- 新增 `CannonProgress`：读写账号升级等级与积分（本地 `PlayerPrefs` + 服务端 `user` 模块）。
- `CannonGameController.StartGame` 读取当前等级，应用到开火冷却、炮弹伤害、防线初始护盾/防御值。
- 把现有 `_lives`（int=3）替换为防线状态 `_shield` / `_defenseHp`；`OnZombieReachedBottom` 改为对防线造成伤害并判负。
- 升级页 UI（MCP 搭建）：4 条线的当前等级 / 下一级数值 / 升级消耗 / 积分余额。

### 4.3 敌人系统：丧尸 + 怪物

**需求（已确认）**：敌人分**丧尸**与**怪物**两大类。
- 丧尸 3 类：普通、快速、坦克。
- 怪物 3 种 + 1 个 Boss。**怪物与 Boss 均无特殊技能**，仅在关卡推进中**血量与护甲（防御值）递增**——这正是玩家需要升级大炮（伤害/射速）与防线（护盾/防御值）的驱动力。

**敌人属性**：每个敌人有 **血量 HP**、**护甲 Armor（减伤）**、**攻击力 AttackPower** 与 **攻击间隔 AttackInterval**（到达防线后每隔该时间对防线造成一次 AttackPower 伤害，直到被消灭）。炮弹实际伤害 = `max(1, 炮弹伤害 - Armor)`，Boss/高级怪护甲高，逼迫玩家升级伤害。

**怪物 3 种差异化（已确认）**：在**外观体型、血量、移速**上均有梯度差异（例如：小型敏捷低血、中型均衡、大型高血慢速），数值全部由 `EnemyConfig` 配置；Boss 为最高血量/护甲/攻击力的首领。

**落地动作**：
- 术语：展示文案、标签（`Zombie.GetLabel`）改为「丧尸 / 怪物 / Boss」；代码类名 `Zombie`/`ZombieSpawner` 保留以减少改动量（仅注释说明）。
- 分类：新增 `enum EnemyCategory { Zombie, Monster, Boss }`；具体种类由配置表 `id` 区分（普通/快速/坦克 + 3 怪物 + Boss）。
- 敌人数值（HP/Armor/速度/分值/体型/对防线伤害/贴图）全部由 `EnemyConfig` 驱动，替换 `Zombie.Configure` 硬编码。
- 关卡难度系数作用于敌人 HP 与 Armor（随关卡递增），速度可微调。
- 战斗系统复用：刷新、点选/射线命中、受击（`TakeHit` 改为按伤害扣血并计护甲减伤）、击杀爆炸均已具备，改为读配置。

### 4.4 限时挑战模式

**需求**：60 秒倒计时、刷分、难度递增。

**现状**：已实现 60s 计时、波次/阶递增、刷新间隔随波次收紧、敌人种类随波次变强。

**落地动作**：
- 抽出难度参数到配置（刷新间隔曲线、各波敌人概率、速度系数），便于调参。
- 接入升级加成（射速/伤害/防线在本模式同样生效）。
- 结算接入评级与积分（见 4.7 / 4.8）。

### 4.5 关卡模式（25 关）

**需求**：排列好不同丧尸/怪物的关卡，对应难度系数，初步 25 关。

**落地动作**：
- 新增 `LevelConfig`（ScriptableObject，25 条）：
  - `levelId`、`难度系数 difficulty`、`敌人编排 waves[]`（每波：敌人类型 + 数量 + 出场间隔 + 速度系数）、`通关条件 = 全歼编排内所有敌人`、`星级阈值（按得分评定）`。
- 新增关卡流程：`CannonGameController` 增加 `GameMode { TimeChallenge, Level }`；关卡模式下不用 60s 计时，按 `LevelConfig` 驱动 `ZombieSpawner` 出怪，**编排全部出完且场上敌人清空即通关**（防线被攻破则失败）。
- `ZombieSpawner` 增加「编排驱动」模式（按脚本出怪），与现有「随机波次」模式并存。
- 选关 UI（MCP 搭建）：25 关网格、解锁状态（通关解锁下一关）、星级显示；解锁/星级存本地 + 服务端。
- 难度系数统一作用于敌人血量/护甲/速度/刷新密度，保证 1→25 平滑递增。
- **章节化**：25 关分为 5 章（每章 5 关），每章末为 **Boss 关**，按章切换主题与敌人外观，制造阶段目标感（借鉴弓箭传说的章节制，详见十一章）。

### 4.6 连击与打击反馈系统

**需求**：连击计数、命中特效触发、震动反馈、得分跳字。

**现状**：连击计数（`_combo`）、跳字（`CannonHUD.ShowHitFeedback` `+分数`）、爆炸（`ShowExplosion`）、震动（`Handheld.Vibrate`）、连击放大动画（`Pulse`）均已具备。

**落地动作（增强）**：
- 连击分级反馈：达到 x3/x5/x10 触发不同特效与文案（如「连击！」「狂暴！」）。
- 命中特效分级：普通命中 / 击杀 / 击杀怪物（更大爆炸 + 屏震）。
- 震动分级：命中轻震、击杀中震、漏怪重震（Android 真机）。
- 得分跳字在命中位置弹出并上飘渐隐（当前在固定 HUD 位，建议改为世界坐标飘字）。

### 4.7 结算与分享

**需求**：得分评级、再来一局、分享成绩到微信。

**现状**：`EndGame` 显示分数/击杀/命中率；`Retry` 按钮已绑定重开。

**落地动作**：
- **评级**：按分数（或星级）映射 S/A/B/C/D，结算页展示评级图标 + 文案。
- **再来一局**：复用现有 `Retry`。
- **微信分享**：结算页「分享」按钮 → 截图当前结算 → 调微信分享 SDK（需接入，预留 `IShareService`）。
- **看视频三倍积分**：结算页「看视频领 3 倍」按钮 → 激励视频 SDK（预留 `IRewardedAdService`）→ 播放完成回调里把本局积分 ×3 入账。
- **防线破复活（可选）**：防线被攻破时弹出「看视频复活」，恢复约 50% 防线继续本局，每局限 1 次（降挫败 + 变现，详见十二章）。

### 4.8 积分系统与经济设计

**需求（已确认）**：通关获积分；看视频得三倍积分。积分用于局外永久升级。下列为**参考初版数值**（对标同类塔防/割草手游的养成节奏），可在调试中平衡微调。

**积分获取**：
- 限时挑战：`基础积分 = floor(得分 / 10)` + 评级奖励（S +500 / A +300 / B +150 / C +50 / D +0）。
- 关卡模式：`通关奖励 = 关卡号 × 50`；`首通额外 = 关卡号 × 100`（仅第一次通关）；星级奖励 3 星 +200 / 2 星 +100 / 1 星 +0。
- **看视频三倍**：结算时观看激励视频，把**本局应得积分整体 ×3**（即额外补 2 倍），每局限一次。

**升级消耗**（每条线 1→10 级，升到下一级所需积分，约 ×1.7 递增）：

| 目标等级 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 | 10 |
|---|---|---|---|---|---|---|---|---|---|
| 消耗 | 200 | 350 | 600 | 1000 | 1700 | 2900 | 4900 | 8300 | 14000 |

- 四条线各自独立按此表升级（可按线微调倍率，例如防线类略贵 1.2×）。

**数值上限参考**（1→10 级）：

| 升级线 | Lv1 | Lv10 |
|---|---|---|
| 射速（开火间隔 s） | 0.42 | 0.15 |
| 伤害 | 1 | 12 |
| 护盾 | 100 | 1000 |
| 防御值 | 200 | 2000 |

**存储**：本地 `PlayerPrefs` 即时缓存 + 服务端 `user` 模块持久化与校验；升级等级、积分、关卡星级同步服务端。激励视频奖励建议服务端校验入账，防止刷分作弊。

## 五、数据结构与配置

建议统一用 ScriptableObject 配置（与项目现有 `Resources/CombatAssets.asset` 模式一致），便于策划调参、热改。

### 5.1 敌人配置 `EnemyConfig`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一标识 |
| `displayName` | string | 展示名（普通丧尸/快速丧尸/坦克丧尸/怪物） |
| `category` | EnemyCategory | Zombie / Monster |
| `maxHp` | int | 血量 |
| `armor` | int | 护甲（减伤），怪物/Boss 较高 |
| `speed` | float | 基础下落速度 |
| `score` | int | 击杀分值 |
| `size` | float | 体型 |
| `attackPower` | int | 到达防线后每次攻击对防线的伤害 |
| `attackInterval` | float | 攻击间隔（秒） |
| `sprite` | Sprite | 贴图（替换色块占位） |

### 5.2 大炮升级配置 `CannonUpgradeConfig`

| 字段 | 类型 | 说明 |
|---|---|---|
| `track` | UpgradeTrack | FireRate / Damage / Shield / Defense（四条线） |
| `levels[]` | LevelEntry | 每级：`value`（数值）、`cost`（升级所需积分） |
| `maxLevel` | int | 上限（初版 10 级） |

- 射速 `value` → `ManualFireCooldown`；伤害 `value` → 炮弹 `Damage`；护盾 `value` → 防线初始护盾上限；防御值 `value` → 防线初始防御生命上限。

### 5.3 关卡配置 `LevelConfig`（25 条）

| 字段 | 类型 | 说明 |
|---|---|---|
| `levelId` | int | 1~25 |
| `difficulty` | float | 难度系数（作用于血量/速度/密度） |
| `waves[]` | WaveEntry | 每波：`enemyId` + `count` + `interval` + `speedMul` |
| `clearCondition` | enum | 全歼 / 存活到编排结束 |

## 十、设计决策（已确认）与剩余开放项

**已确认**：
1. **升级 = 局外永久养成**（用积分买等级，账号存档，非局内临时强化）。
2. **防线 = 城墙式防御线**，含 **护盾 + 防御值** 两层：先扣护盾，护盾归零后扣防御值，防御值归零即对局失败；两者均可独立升级。**取代原「3 条命」**。
3. **敌人**：丧尸 3 类（普通/快速/坦克）+ 怪物 3 种 + Boss；**无特殊技能**，仅血量与护甲随关卡递增（驱动玩家升级大炮与防线）。
4. **关卡通关条件 = 全歼**编排内所有敌人；共 25 关。
5. **积分经济**：见 4.8（参考初版数值，可调）；看视频奖励为**本局积分整体 ×3**。
6. **关卡模式**：无 60s 计时，沿用防线机制；限时挑战保留 60s。

**已确认（补充）**：
7. **数据存储**：以本地 `PlayerPrefs` 为主先落地；搭功能模块时**把服务端同步接口逻辑一并写好打通**（双写/后续切换不返工）。
8. **美术素材与广告/微信 SDK**：由客户方提供，代码侧预留接入点（接口 + 占位实现）。
9. **防线受击表现**：敌人到达防线后**停驻并持续攻击**防线，被消灭后才消失。
10. **怪物差异化**：3 种怪物在**外观体型、血量、移速**上均有差异；Boss 为最高数值首领。

**剩余开放项**：
- A. 升级/敌人/经济等数值与平衡，需真机体验后持续微调（常规迭代，不阻塞开工）。

## 十一、市面同类游戏参考与借鉴

调研了几款同类爆款，提炼可借鉴点（**核心框架不变**，下列为补充/优化建议）：

| 对标产品 | 核心机制 | 可借鉴点 |
|---|---|---|
| 《向僵尸开炮》 | 射击 Roguelike：瞄准打僵尸 + 大炮/英雄养成 + 章节关卡（[来源](https://www.ldmnq.com/app/8293/)） | 与本项目最接近；其玩家普遍反馈「升级慢、重复刷」是痛点，我们养成曲线要保证**每局可感知成长** |
| 弓箭传说 Archero | 章节制（难度递增 + 章末 Boss）、Die-and-Retry、技能堆叠（穿透/弹射/多重）、金币+卷轴升级（[来源](https://archero.fandom.com/wiki/Category:Chapters)） | 关卡**章节化 + 章末 Boss**；中后期可选**弹种/技能堆叠**增加深度 |
| Survivor.io / 幸存者类 | 自动攻击、玩家只管走位与选择；武器进化、套装链式触发（[来源](https://survivor-io.io/)） | 印证「**自动开火 + 玩家专注选目标/养成**」方向；武器进化可作扩展 |
| 植物大战僵尸 | 经典塔防：敌人分路推进、单位克制 | 敌人**分路/分波**与**类型克制**的关卡编排手法 |

> 上述依据公开资料整理并改写，仅作设计参考。

**提炼的设计原则**：
1. **操作极简、深度在养成与策略**——契合体感健身/休闲定位，点选即可上手。
2. **关卡章节化**——25 关分 5 章、章末 Boss、换主题与外观，制造阶段目标感。
3. **可感知成长**——前期升级便宜、反馈快；避免「长时间刷却无成长」的疲劳（《向僵尸开炮》被诟病点）。
4. **失败不劝退**——提供看广告复活/续防线，降低挫败并兼顾变现。

## 十二、留存与变现设计（参考同类）

> 免费 + 广告为主，契合体感健身的休闲定位；广告以**奖励型**为主、不强插，避免破坏体验。

**留存机制**：
- **每日任务**：击杀 N 个、通关 X 关、升级 1 次等 → 奖励积分。
- **每日签到** + **限时挑战每日刷新**（配合排行榜冲榜）。
- **关卡首通额外奖励** + **三星目标**，提供重复挑战动机。
- **中长期目标**：解锁新章节、点满某条升级线、收集怪物图鉴。

**变现（激励视频为主）**：

| 点位 | 设计 | 说明 |
|---|---|---|
| 结算积分翻倍 | 看视频本局积分 ×3（已定） | 核心点位 |
| 防线破复活 | 看视频复活一次、恢复约 50% 防线继续本局 | 降挫败 + 变现，每局限 1 次 |
| 免费宝箱 | 看视频开额外积分宝箱 | 每日限次 |
| 限时双倍 | 看视频激活 N 分钟积分双倍 buff | 提升单次时长 |
| 升级折扣 | 看视频减免一次升级消耗 | 可选 |

> 健身定位下广告频次需克制：优先「对玩家有利」的奖励型广告（复活/翻倍/宝箱），避免对局中插屏打断。

## 十三、养成深度扩展（可选，中后期）

> 4 条主线（射速/伤害/护盾/防御值）满足首期。以下为中后期可选的深度扩展，借鉴割草/Roguelike 类武器成长，**按需排期，不影响首期开工**。

- **暴击**：暴击率 / 暴击伤害两条副线。
- **多重 / 分裂弹**：一次发射多发，或命中后分裂，提升清线能力。
- **穿透弹**：一发贯穿同方向多个敌人（呼应 Archero 穿透箭）。
- **溅射 / 范围伤害**：命中产生小范围爆炸，对密集敌群高效。
- **特殊炮弹**：冰冻（减速）、火焰（持续伤害）、清屏炸弹——可做主动技能或局内 Roguelike 三选一。
- **大炮皮肤 / 外观**：积分或付费解锁，纯外观，呼应产品方案的皮肤系统。

> 这些扩展会显著加深策略与长线追求，建议在双模式与基础养成稳定后（P3 之后）评估接入。
