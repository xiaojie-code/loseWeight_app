# 大炮射击体感版 - 实现方案

> 基于当前项目 ML Kit 姿态检测方案，新增"大炮射击"游戏模块

---

## 一、核心玩法

- 用户正对手机前置摄像头
- 一只手控制瞄准方向（手上下左右移动 → 准星跟随）
- 手快速往后拉（手腕靠近肩膀方向回缩）→ 触发炮弹发射
- 炮弹从屏幕底部大炮位置飞向准星位置
- 僵尸从屏幕各方向出现并向中心移动，被炮弹命中即消灭
- 限时 60 秒，击杀越多得分越高

### 参考游戏

| 游戏 | 借鉴点 |
|---|---|
| Angry Birds | "拉后发射"的操作手感，拉的幅度可以影响力度 |
| Zombie Waves | 波次递增难度，僵尸种类多样 |
| 植物大战僵尸 | 僵尸从远处走来的视觉节奏 |
| 打靶类游戏 | 准星跟随 + 命中反馈 |

## 二、操作设计

### 2.1 瞄准

```
ML Kit 输出: 右手手腕 x/y ∈ [0, 1]
  ↓ 镜像翻转
屏幕准星位置: screenX = (1 - x) * Screen.width
              screenY = (1 - y) * Screen.height
```

准星实时跟随手腕位置，用户移动手臂即可瞄准。

### 2.2 发射触发（往后拉）

检测逻辑：
- 每帧计算"手腕到肩膀的 2D 距离"
- 当距离**突然减小**（手往回缩）且减小幅度 > 阈值 → 判定为"拉后发射"
- 类似弹弓：手往后拉 = 蓄力，松手/到位 = 发射

```
当前帧距离 - 上一帧距离 < -PULL_THRESHOLD → 触发发射
```

**为什么用"往后拉"而不是"往前推"**：
- 往后拉时手腕靠近身体，ML Kit 的 x/y 变化更明显（手腕从远处回到肩膀附近）
- 符合"拉弹弓/拉炮绳"的直觉
- 不会和瞄准动作冲突（瞄准是慢速移动，发射是快速回缩）

### 2.3 操作流程

```
1. 手伸出 → 准星出现在手指向的位置
2. 慢慢移动手 → 准星跟随，对准僵尸
3. 手快速往后拉（像拉弹弓一样）→ 炮弹发射
4. 手再伸出 → 继续瞄准下一个目标
```

## 三、游戏机制

### 3.1 僵尸系统

| 僵尸类型 | 颜色 | 血量 | 速度 | 分数 | 出现时机 |
|---|---|---|---|---|---|
| 普通僵尸 | 绿色 | 1 | 慢 | 100 | 全程 |
| 快速僵尸 | 紫色 | 1 | 快 | 150 | 20s 后 |
| 胖僵尸 | 棕色 | 2 | 很慢 | 200 | 30s 后 |
| Boss 僵尸 | 红色 | 5 | 极慢 | 500 | 50s 后 |

### 3.2 波次系统

| 时间段 | 生成频率 | 同屏最大数 | 特殊 |
|---|---|---|---|
| 0-15s | 1.5s/个 | 3 | 只有普通僵尸 |
| 15-30s | 1.2s/个 | 5 | 加入快速僵尸 |
| 30-45s | 0.9s/个 | 6 | 加入胖僵尸 |
| 45-60s | 0.7s/个 | 8 | 加入 Boss |

### 3.3 僵尸行为

- 从屏幕上半部分随机位置出现
- 缓慢向屏幕下方移动（模拟"向玩家走来"）
- 到达屏幕底部 = 扣一条命
- 被击中 = 播放死亡动画 + 消失
- 胖僵尸/Boss 需要多次命中

### 3.4 计分

| 事件 | 分数 |
|---|---|
| 击杀普通僵尸 | +100 |
| 击杀快速僵尸 | +150 |
| 击杀胖僵尸 | +200 |
| 击杀 Boss | +500 |
| 连续命中（不 miss）| 连击加成 ×1.5 |
| 僵尸到达底部 | 扣一条命 |

### 3.5 生命系统

- 3 条命
- 僵尸到达屏幕底部扣一条命
- 命用完 = 游戏结束（不等 60 秒）

## 五、核心算法

### 5.1 "往后拉"发射检测

```csharp
// 状态机：Armed → Pulling → Fired → Cooldown → Armed
enum FireState { Idle, Armed, Pulling, Cooldown }

// Armed 条件：手腕到肩膀距离 > ARM_THRESHOLD（手伸出去了）
// Pulling 条件：距离开始减小（手开始回缩）
// Fired 条件：回缩速度 > PULL_SPEED_THRESHOLD 且累计回缩量 > PULL_DISTANCE

每帧：
  float currentDist = Vector2.Distance(wrist.xy, shoulder.xy);
  
  switch (state):
    case Idle:
      if (currentDist > ARM_THRESHOLD) state = Armed;
      break;
    case Armed:
      if (currentDist < lastDist - 0.01) // 开始回缩
        state = Pulling; pullStartDist = lastDist;
      break;
    case Pulling:
      float pullAmount = pullStartDist - currentDist;
      float pullSpeed = (lastDist - currentDist) / dt;
      // 蓄力反馈：准星缩小 + 变亮
      ShowChargeFeedback(pullAmount / PULL_DISTANCE);
      if (pullAmount > PULL_DISTANCE && pullSpeed > PULL_SPEED_THRESHOLD)
        Fire(); state = Cooldown; cooldownEnd = now + 0.6;
      else if (currentDist > lastDist + 0.02) // 手又伸出了，取消
        state = Armed;
      break;
    case Cooldown:
      if (now > cooldownEnd) state = Idle;
      break;
```

**防误触设计**：
- 必须先 Armed（手伸出）才能触发，慢速移动手不会误触
- Pulling 状态下如果手又伸出去了（用户只是调整位置），自动取消回到 Armed
- 冷却 0.6 秒防止连发
- 前 8 帧不检测（等姿态稳定）
- 需要累计回缩量 + 速度双重条件

**参数建议**：
- ARM_THRESHOLD = 0.15（归一化，手腕到肩膀距离 > 15% 画面宽度）
- PULL_DISTANCE = 0.06（回缩 6% 画面宽度触发）
- PULL_SPEED_THRESHOLD = 0.3（归一化/秒）

### 5.2 炮弹飞行与碰撞

```
发射时：
  起点 = 屏幕底部大炮位置
  终点 = 当前准星位置（发射瞬间锁定）
  飞行时间 = 0.3 秒

飞行中：
  每帧线性插值更新位置
  每帧检测炮弹当前位置是否和任何僵尸碰撞（穿透检测）
  命中 → 僵尸扣血 + 炮弹消失 + 爆炸特效
  到达终点未命中 → 炮弹消失 + miss 反馈

命中判定：
  炮弹位置和僵尸中心距离 < (僵尸半径 + 炮弹半径)
  优先命中距离炮弹最近的僵尸（避免穿透多个）
```

### 5.3 瞄准平滑

```csharp
// EMA 平滑，避免准星抖动（20fps 下关键点会跳）
smoothX = smoothX * 0.65 + rawX * 0.35;
smoothY = smoothY * 0.65 + rawY * 0.35;

// 蓄力时准星视觉反馈
if (state == Pulling)
{
    float chargeRatio = pullAmount / PULL_DISTANCE; // 0~1
    crosshairScale = Mathf.Lerp(1.0, 0.5, chargeRatio); // 缩小
    crosshairColor = Color.Lerp(normalColor, chargeColor, chargeRatio); // 变亮
}
```

### 5.4 僵尸行为细节

```
移动：
  从生成位置匀速向屏幕底部移动
  速度按类型不同（普通 80px/s，快速 150px/s，胖 50px/s）

到达底部警告：
  距离底部 < 20% 屏幕高度时开始红色闪烁
  到达底部 → 扣命 + 消失 + 屏幕边缘红色闪光

受击：
  普通/快速僵尸：1 发即死
  胖僵尸：2 发，第一发变色 + 减速
  Boss：5 发，每发缩小一点 + 变色加深
```

### 5.5 瞄准手选择

```
策略：取两只手中距离肩膀更远的那只作为瞄准手
  leftDist = distance(leftWrist, leftShoulder)
  rightDist = distance(rightWrist, rightShoulder)
  aimHand = leftDist > rightDist ? left : right

好处：
  - 用户自然伸出哪只手就用哪只手
  - 不需要设置"左撇子/右撇子"
  - 两只手交替使用也行（增加运动量）
```

## 九、风险点与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| "往后拉"和"瞄准移动"混淆 | 瞄准时误发射 | 状态机设计：必须先 Armed 再 Pulling，慢速移动不触发 |
| 准星抖动 | 瞄不准 | EMA 平滑 + 适当放大命中半径 |
| 僵尸太小打不中 | 体验差 | 僵尸尺寸 60-100px + 命中半径额外 +20px |
| 20fps 姿态检测延迟 | 准星跟手慢 | 可接受（瞄准是持续动作，不需要瞬时响应）|
| 左右手映射反 | 准星方向不对 | 自动选择伸出更远的手，不依赖固定左右 |
| 用户不知道怎么发射 | 上手难 | 开局前加 3 秒教学动画（手伸出 → 拉回 → 发射）|
| 没有触觉反馈 | 打击感弱 | 发射/命中时手机震动（Handheld.Vibrate）|

## 十一、教学引导

首次进入大炮射击时显示 3 秒教学提示：

```
画面中央显示：
  🖐️ → 伸出手瞄准
  ✊ ← 快速拉回发射
  
底部文字："伸手瞄准，拉回发射"
3 秒后自动消失，开始倒计时
```
