# 减肥拳击对战游戏 - 项目进度报告

> 更新时间：2026-05-25

---

## 一、项目概况

Unity 手机端拳击对战游戏，玩家通过前置摄像头 + ML Kit 姿态识别控制角色出拳/防御/闪避，和 AI 对手打拳击。

技术栈：Unity 2022 + C# + Google ML Kit Pose Detection (Bundled) + NestJS 服务端

---

## 二、今日完成的事

1. **去掉了对局内的屏幕调试日志**（绿字）—— ScreenDebugLog 默认 Enabled=false，再玩一局也不会残留旧文本
2. **去掉了校准引导页**（"请站在摄像头前 1.5 米处"那个页面）—— 点对战直接进入对局，不卡在校准页
3. **修复了玩家模型被 AI 打时的抖动**—— 删掉了 CharacterAnimationDriver 里"玩家受击播 Duck 动画"的逻辑，现在玩家模型只响应姿态识别事件
4. **出拳检测做了多版调参**—— 引入准备-触发状态机、噪声门槛、EMA 平滑基线、出拳后禁用闪避等机制
5. **MLKit 检测帧率从 14fps 提到 20fps**（间隔 70ms → 50ms），降低端到端延迟

## 四、未完成 / 待优化

1. **出拳左右映射可能还是反的** —— 前置摄像头未镜像图像送 ML Kit，LEFT_WRIST 实际对应用户右手。已调过多次，需要真机再确认一次
2. **右拳触发率低于左拳** —— 上次测试右拳无法触发，怀疑是 arm 判定不对称或者基线偏了，需看 logcat 数据定
3. **闪避/下蹲暂时关闭**（`EnableDodgeDetection = false`）—— 之前闪避误触太严重，等出拳稳定后再开
4. **PoseOverlay 关键点位置和画面人体不对齐** —— 预览 RawImage 做了 -90° 旋转 + 水平镜像，但 Overlay 画点时没做同样变换
5. **再玩一局时摄像头/检测可能丢失** —— CleanupCombatResources 清理后重建的时序偶尔不对
6. **在线对战/房间匹配还没真正对接服务端** —— 目前只有人机模式可用
7. **音效/打击特效还是空的** —— AudioManager 有框架但没有实际音频资源

## 五、当前可能存在的问题

- **延迟感**：端到端（出拳 → 模型响应）大概 200-300ms，算是能接受但还有优化空间（主要瓶颈在 JPEG 编码 + ML Kit 推理）
- **ActionDetector 版本混乱**：git 仓库里是最初的"打分制"复杂版本，本地调试时改过多次简化版又改回来，需要确认当前编译用的到底是哪一版
- **摄像头预览 RawImage 的旋转/镜像是写死的** rot=-90 + scaleX=-1，换手机可能不对（有些手机 rotationAngle 不是 270）
- **ML Kit Java 单例在再玩一局时调用 reinit()**，如果 close() 没调用干净可能内存泄漏
- **CalibrationPage 虽然跳过了但代码还在**，不影响运行，但如果以后想重新加回来需要确保 ActionDetector 的内部 20 帧自校准和 UI 校准不冲突
