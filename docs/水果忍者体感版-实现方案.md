# 水果忍者体感版 - 实现方案

> 基于当前项目 ML Kit 姿态检测方案，新增一个"水果忍者"游戏模块

---

## 一、核心玩法

- 用户正对手机前置摄像头，ML Kit 识别双手手腕位置
- 屏幕上两只手对应位置各显示一把刀（跟随手移动）
- 水果从屏幕边缘/底部抛出（抛物线轨迹）
- 用户挥手 → 刀的运动轨迹碰到水果 = 砍中得分
- 炸弹出现时不能砍（砍到扣分/扣命）
- 限时模式或生命模式，最终计总分

## 三、核心逻辑设计

### 3.1 手部位置映射

```
ML Kit 输出: 手腕 x/y ∈ [0, 1]（归一化图像坐标）
  ↓
屏幕坐标: screenX = (1 - x) * Screen.width   // 前置镜像翻转
           screenY = (1 - y) * Screen.height   // y轴翻转（ML Kit y向下）
  ↓
刀的 UI 位置: 每帧更新 RectTransform.anchoredPosition
```

注意：前置摄像头镜像问题和拳击一样，第一次进入时让用户看屏幕上的刀确认方向对不对。

### 3.2 轨迹碰撞检测（核心算法）

不能只判断"当前帧手在水果位置上"，因为 20fps 时快速挥手两帧间手移动了半个屏幕，水果可能被穿过但没检测到。

**方案：线段-圆碰撞检测**

```
每帧：
  lineStart = 上一帧手的屏幕坐标
  lineEnd   = 当前帧手的屏幕坐标
  
  对每个活跃水果：
    center = 水果当前屏幕坐标
    radius = 水果碰撞半径（比贴图大 20%，降低难度）
    
    if LineSegmentIntersectsCircle(lineStart, lineEnd, center, radius):
      水果被切中
```

这样即使手移动很快跳了很远，只要轨迹经过水果就判定命中。

### 3.3 水果生成与飞行

```
生成逻辑：
  - 从屏幕底部/两侧随机位置生成
  - 给一个初始向上的速度（+ 随机水平分量）
  - 受重力影响做抛物线
  - 飞出屏幕顶部后回落，落到屏幕底部以下销毁（未切中 = miss）

难度曲线：
  - 前 30 秒：单个慢速水果，间隔 1.5s
  - 30-60 秒：2-3 个同时，间隔 1.0s，速度加快
  - 60 秒后：4-5 个同时，混入炸弹，速度继续加快
```

### 3.4 计分系统

| 事件 | 分数 |
|---|---|
| 切中普通水果 | +10 |
| 连击（0.5s 内连续切中）| +5 × 连击数 |
| 切中特殊水果（低频出现）| +50 |
| 切到炸弹 | -30 + 扣一条命 |
| 水果落地未切中 | 扣一条命 |

### 3.5 游戏模式

- **经典模式**：3 条命，漏掉水果或砍到炸弹扣命，无限时间
- **限时模式**：60 秒，尽可能多得分，漏水果不扣命
- **禅模式**：无炸弹无惩罚，纯练习（调试用）

## 四、UI 与视觉设计

### 4.1 屏幕布局

```
┌─────────────────────────────┐
│  分数: 1280    连击: x5     │  ← 顶部 HUD
│  ❤❤❤          ⏱ 45s       │
├─────────────────────────────┤
│                             │
│     🍎         💣          │  ← 水果/炸弹飞行区域
│          🍊                │
│    🔪              🔪      │  ← 双手位置的刀（跟随手移动）
│         ～～～～            │  ← 刀痕轨迹（拖尾特效）
│                   🍋       │
│                             │
├─────────────────────────────┤
│  ▓▓▓▓▓▓ 水果生成区 ▓▓▓▓▓▓  │  ← 底部（水果从这里飞出）
└─────────────────────────────┘
```

### 4.2 视觉元素

| 元素 | 实现方式 |
|---|---|
| 刀 | 简单 Sprite（刀片图标），跟随手腕坐标 |
| 刀痕 | LineRenderer 或 Trail，保留最近 8 帧位置连线，渐隐消失 |
| 水果 | 2D Sprite（圆形贴图），用 Unity UI Image 即可 |
| 切割效果 | 水果分成两半 + 粒子喷射（果汁颜色）|
| 炸弹 | 黑色圆球 + 导火线动画 |
| 背景 | 纯色渐变 / 可选显示摄像头画面（半透明叠加）|

### 4.3 摄像头画面处理

两种方案供选择：
- **方案 A**：摄像头全屏半透明作为背景，用户能看到自己 + 水果叠加在上面（沉浸感强）
- **方案 B**：摄像头缩小在角落（和拳击一样），游戏画面独立（更清爽）

推荐先做方案 B（和拳击一致），后续再试方案 A。

## 五、与现有系统的集成

### 5.1 GameState 新增

```csharp
// Core/GameState.cs 新增：
FruitNinja,        // 水果忍者模式
FruitNinjaEnd,     // 水果忍者结算
```

### 5.2 主页入口

在 MainMenuPage 加一个"水果忍者"按钮：
```
点击 → GameManager.ChangeState(GameState.FruitNinja)
    → UIManager 隐藏所有页面
    → FruitGameController 启动（类似 GameController 对 Combat 的处理）
```

### 5.3 姿态检测复用

FruitGameController 不经过 ActionDetector（因为不需要动作分类），直接监听 MLKitPoseProvider.OnPoseFrame：

```csharp
void OnPoseFrame(PoseFrame frame)
{
    var lw = frame.GetLandmark(PoseLandmarkIndex.LeftWrist);
    var rw = frame.GetLandmark(PoseLandmarkIndex.RightWrist);
    
    // 直接用 x/y 坐标更新刀的位置
    _sliceDetector.UpdateHands(lw.Position, rw.Position, lw.Confidence, rw.Confidence);
}
```

### 5.4 不影响拳击模块

- 新模块完全独立（`Assets/Scripts/FruitNinja/` 目录下）
- 共用 MLKitPoseProvider 但不共用 ActionDetector
- 通过 GameState 区分当前是哪个游戏模式

## 七、风险点与应对

| 风险 | 影响 | 应对方案 |
|---|---|---|
| 左右手映射反了 | 刀跟手方向不对 | 第一次进入让用户挥手确认，加一个镜像开关 |
| 20fps 快速挥手丢帧 | 轨迹不连续，穿过水果没判中 | 线段碰撞检测 + 扩大碰撞半径 |
| 手腕置信度偶尔下降 | 刀突然跳位 | 置信度 < 0.3 时冻结上一帧位置 |
| 发热/掉帧 | ML Kit + 水果特效同时跑 | 控制同屏水果数 ≤ 8，粒子特效做对象池 |
| 玩家手臂酸累 | 体验问题 | 每局 60-90 秒，局间休息提示 |

## 八、后续扩展方向

- 多种道具（武士刀、光剑、拳套）—— 换皮 + 不同轨迹宽度
- 双人模式：屏幕分左右区域，两人各控制一只手
- 排行榜接入服务端（复用现有 NestJS rank 模块）
- 特殊关卡：只切特定颜色水果、节奏模式（配合音乐节拍出水果）
- 运动数据统计：挥手次数、消耗卡路里估算
